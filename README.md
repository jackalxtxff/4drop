# 4drop — интеграция 4tochki ↔ Wildberries / Ozon

Дропшиппинг-прослойка: каталог, остатки и цены тянутся из B2B-кабинета 4tochki,
карточки и остатки уезжают на маркетплейсы, заказы с маркетплейсов возвращаются
поставщику.

## Запуск

```bash
cp .env.example .env
# заполнить SECRETS_KEY и JWT_SECRET:
#   python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
#   openssl rand -hex 32
docker compose up
```

- Интерфейс: http://localhost:5173
- API: http://localhost:8080 (Swagger — `/docs`)
- Первый пользователь создаётся из `BOOTSTRAP_USER_EMAIL` / `BOOTSTRAP_USER_PASSWORD`.

Сервисы: `backend` (FastAPI), `worker` (arq), `db` (PostgreSQL), `redis`, `frontend` (Vite).

## Что уже работает

- Вход по логину/паролю (argon2 + JWT).
- Поставщики — центральная сущность; переключатель активного поставщика в шапке.
  Доступы, каталог, маппинги и заказы изолированы по `supplier_id`.
- «Подключения»: три блока (4tochki / WB / Ozon), проверка подключения по кнопке,
  секреты шифруются (Fernet) и показываются маской. Склады 4tochki подтягиваются
  из `GetWarehouses` после успешной проверки; у каждого виден срок логистики.
- «Товары»: виртуализованная таблица, серверные фильтры и пагинация, множественный
  выбор, кнопка «Обновить каталог из 4tochki», прогресс фоновой задачи.
- Выгрузка каталога в фоне: `GetFindTyre`/`GetFindDisk` постранично + `GetGoodsInfo`
  пачками; цены и остатки складываются в разрезе складов.

## Ключевые решения

**4tochki — это SOAP/WCF, не REST.** См. `docs/4tochki-api.md`: там снятые с WSDL
сигнатуры. Токена/сессии нет — логин и пароль уходят параметрами в каждый вызов,
поэтому пароль шифруется в БД и расшифровывается только в памяти воркера.

**Остатки и цены хранятся по складам** (`product_stocks`), а не агрегатом. Смена
набора выбранных складов пересчитывает агрегаты без перевыкачки каталога.

**Склады с долгой логистикой — риск для SLA.** При FBS со своего склада `logisticDays`
напрямую съедает срок сборки на маркетплейсе, поэтому склады выбираются явно, и
«ни одного склада не выбрано» означает нулевые остатки, а не «все склады».

**Заказ в 4tochki идёт через `CreateOrder`** (доставка на наш склад), потому что схема —
FBS со своего склада. `CreateMarketplaceOrder` (доставка конечному покупателю) в клиенте
тоже реализован — он понадобится при переходе на realFBS/DBS.

## Открытые вопросы

- **Маркировка «Честный знак».** В API есть `SetMarkCodeList`, `CheckInventoryMarkCode`,
  а `CreateOrderResult.goods[].marking_is_required` прямо помечает позиции, требующие
  маркировки. Но кто выводит товар из оборота при продаже через МП — вопрос договора
  с 4tochki. Блокирующий на пути в прод.
- Лимиты запросов 4tochki в WSDL не описаны — уточнить у поставщика; сейчас батч 200.
